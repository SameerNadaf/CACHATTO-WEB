No need for VDI, DaaS, or remote desktop!
A data-less client that allows you to access your company network via a secure VPN.

This data-less client prevents information leaks by protecting data in a "secure container area" created when the PC starts up and deleting data at the end of the workday. It features a unique VPN connection function that does not expose VPN equipment to the internet, providing secure access to cloud/on-premises environments.

Are you experiencing any of these problems?

We are using/considering using VDI or DaaS, but...

Problems
~
  • ■ Implementing and operating a virtual desktop environment incurs significant costs.
  • ■ Each user needs two PC environments (one for access and one for the desktop environment they will be connecting to).
  • ■ It is highly dependent on the communication environment, the quality of web conferences is unstable, and cameras cannot be used.
→
With Secure Container AD
  • ✔ It eliminates the need to build expensive and large-scale virtual desktop environments, and costs are up to 1/10th compared to VDI/DaaS.
  • ✔ No PC environment is required for connection; one PC per person is sufficient to set up a work environment.
  • ✔ Local breakout reduces bandwidth load, allowing for comfortable meetings using the camera, microphone, and internet connection of your local PC.

I'm using remote desktop, but...

Problems
~
  • ■ Each person needs two PCs (one for access and one for connecting to the target PC in the office).
  • ■ Office PCs need to be kept powered on at all times.
  • ■ It is highly dependent on the communication environment, the quality of web conferences is unstable, and cameras cannot be used.
→
With Secure Container AD
  • ✔ No additional PC is required for connection; one PC per person is sufficient to set up the work environment.
  • ✔ No connected PC is required, reducing the burden of PC operation in the office, such as troubleshooting PC startup and power outage response.
  • ✔ Local breakout reduces bandwidth load, allowing for comfortable meetings using the camera, microphone, and internet connection of your local PC.

I'm using a company-issued PC and a VPN, but...

Problems
~
  • ■ The files remain on the PC as they are.
  • ■ VPN servers exposed on the DMZ are vulnerable to cyberattacks and have a high operational burden in terms of security.
→
With Secure Container AD
  • ✔ Saved files are encrypted and deleted upon termination.
  • ✔ Our VPN service eliminates the need to expose servers to the DMZ and requires no operation or management by the customer. This minimizes the risk of cyberattacks.

Product Features and Functions

Minimize the risk of external attacks

☠️

No need to expose VPN equipment to the internet, minimizing threats such as ransomware.

Protecting data with virtual disks

Data within the secure container area is encrypted and deleted upon signing out.

Strong authentication features such as Active Directory

Strong authentication is achieved through Active Directory authentication and MFA (multi-factor authentication).

Automatic file synchronization prevents data loss.

✨

Automatically synchronizes to a specified file server or cloud server, and backs up to a specified location.

Work comfortably using local resources

W X P O N

It uses local resources such as apps installed on the device, memory, CPU, and camera.

Work is possible even in an offline environment.

Saved files are temporarily stored locally. This allows for file editing even if the network connection is unstable.

System Configuration Image

Access to the internal network is provided via "CACHATTO One Management Point," a cloud-based service offered by e-Jan Networks. The CACHATTO connector (virtual or physical) for the VPN equipment is installed in the on-premises environment and is not exposed to the internet. The connector uses outbound HTTPS communication.
